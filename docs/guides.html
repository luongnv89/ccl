How to Select & Verify Models Manually

While llmfit automates model selection, you can manually choose and verify models for your hardware. Here's how:

1

Check Your Available Memory

Determine your total available memory for model inference:

  • Mac (unified memory): Total system RAM minus ~4-6GB for OS
  • NVIDIA GPU: VRAM + system RAM (for offloading layers)
  • CPU-only: Total system RAM minus ~8-12GB for OS and applications

Example: M3 Max with 36GB → ~30-32GB available for models

2

Calculate Model Memory Requirements

Estimate memory needed based on model size and quantization:

  • FP16 (full precision): ~2 bytes per parameter
  • Q8 (8-bit quantization): ~1 byte per parameter
  • Q4 (4-bit quantization): ~0.5 bytes per parameter
  • Context buffer: Add 2-4GB for context window

Example: 7B model in Q4 → 7B × 0.5 = 3.5GB + 2GB context = ~5.5GB total

3

Choose Quantization Level

Balance quality vs. memory usage:

  • Q8: Best quality, minimal degradation (~95% of FP16 performance)
  • Q5/Q6: Good balance for most use cases (~90% performance)
  • Q4: Acceptable quality, 2x memory savings (~85% performance)
  • Q3/Q2: Maximum compression, noticeable quality loss (not recommended for coding)

Recommendation: Start with Q4 or Q5 for coding models

4

Test Model Performance

Verify the model runs well on your hardware:

  • Load time: Should load in under 30 seconds
  • First token latency: Under 2-3 seconds for good UX
  • Generation speed: Aim for 10+ tokens/second for interactive coding
  • Memory headroom: Keep 10-20% free memory to avoid swapping

Test command: ccl chat --model <model-name> --benchmark

5

Monitor System Resources

Watch for performance issues during inference:

  • Memory pressure: Check Activity Monitor (Mac) or Task Manager (Windows)
  • Swap usage: If swap increases significantly, model is too large
  • GPU utilization: Should be 70-90% during generation
  • Temperature: Sustained high temps may throttle performance

Red flags: Swap usage >2GB, generation speed <5 tokens/sec, system lag

6

Optimize If Needed

If performance is poor, try these adjustments:

  • Lower quantization: Switch from Q5 to Q4 or Q4 to Q3
  • Smaller model: Drop from 34B to 13B or 13B to 7B
  • Reduce context: Lower max context length (e.g., 4K → 2K)
  • Enable offloading: Use GPU + CPU hybrid inference (if supported)

Goal: Consistent 10+ tokens/sec with <2s first token latency

Pro tip: Start with a smaller model (7B-13B) to verify your setup works, then scale up to larger models once you've confirmed performance is acceptable.
